# Release Plan

## Release Cadence

- Minor release every 2 sprints.
- Patch release as needed for defects and security updates.

## Planned Releases

1. `v0.1.0` (Foundation): governance baseline and initial APIs.
2. `v0.2.0` (Intent Core): MGT validation/versioning complete.
3. `v0.3.0` (Orchestration Core): consultant/auditor loops operational.
4. `v0.4.0` (Integration): GAS + GitHub + GAG roundtrip.
5. `v1.0.0` (Production): full ASDLC governance and release controls.

## Release Entry Criteria

1. Stories and tasks linked to release candidate scope.
2. Required sprint evidence complete for covered sprints.
3. ASDLC bundle fields complete per `07_releases/asdlc-bundle-spec.md`.
4. Blocking CI gates pass; any exceptions formally approved.
