# Sessions and Decision Trace

This folder stores product and engineering session records to preserve decision context.

## Layout

- Templates remain in this folder root.
- Executed session logs are stored under `logs/`.

## Naming Convention

Use `SL-SPR<nn>-YYYY-MM-DD-HHmm-topic.md` for executed logs.

## Required Metadata for Every Session

1. Session ID
2. Date/time
3. Sprint ID
4. Story IDs
5. Decisions made
6. Risks and blockers
7. Next actions
