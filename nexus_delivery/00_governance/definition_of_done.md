# Definition of Done (DoD)

A story is Done when all criteria are met.

## Blocking Criteria

1. Code implemented and merged.
2. Required tests pass.
3. No critical/high findings in security checks.
4. Story acceptance criteria validated by Product Owner.
5. Story ID, task links, and session trace are recorded.
6. Decision record updated when architecture changes.
7. MGT/intent linkage recorded and version updated when needed.

## Advisory Criteria

1. API docs/spec updated when endpoint changes exist.
2. Release notes entry added when user-visible behavior changed.
3. Sprint log and velocity snapshot updated by sprint close.
