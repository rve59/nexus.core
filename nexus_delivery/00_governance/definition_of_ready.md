# Definition of Ready (DoR)

A story is Ready when all criteria are met.

1. Story has ID in format `US-<epic>-<feature>-<seq>`.
2. User value is explicit: "As a ... I want ... so that ...".
3. Acceptance criteria are testable and unambiguous.
4. Dependencies are identified and linked.
5. Estimation is assigned.
6. Target sprint is assigned.
7. Test strategy is specified (unit/integration/contract/workflow/e2e).
8. API contract impacts are described when relevant.
9. MGT impact is specified (none/update/new version).
10. Story references planned test artifacts and traceability links.
11. Dependency record exists in `02_backlog/dependencies.md` when applicable.
12. Refinement decision is captured in `02_backlog/refinement-log/`.

If any criterion fails, the story stays in Backlog Refinement.
